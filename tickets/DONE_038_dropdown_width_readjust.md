# 038: ブキ/SPドロップダウン幅を再調整する

## 種別
UI改善

## 問題
033で `w-44`（176px）に拡大したが、まだ長すぎるか短すぎるかの再検討が必要。

## 分析
text-xs（font-size: 12px）で日本語ラベルを表示する場合:
- 「クマサン印のストリンガー」12文字 × 約12px = 144px + padding
- 「メガホンレーザー5.1ch」半角混在で約130px + padding

select 要素には左右 px-1（4px×2）+ ブラウザ内蔵の矢印アイコン分（約20px）があるため、テキスト表示領域は `w-XX` から約28px を引いた値。

- w-36（144px）: テキスト領域 116px → 12文字の日本語には不足
- w-40（160px）: テキスト領域 132px → ギリギリ
- w-44（176px）: テキスト領域 148px → 余裕あるが幅を取りすぎ

## 修正箇所
- `src/components/Settings/MemoSection.tsx`

## 修正内容
`w-44` を `w-40`（160px）に変更する。最長名がギリギリ収まる幅。フォント環境差で微妙に途切れる可能性はあるが、ブキ・SPは select 要素なので選択前にフルネームがドロップダウンリスト内で確認できるため実用上問題ない。

```tsx
// 変更前
className="w-44 rounded-sm border ..."
// 変更後
className="w-40 rounded-sm border ..."
```

ブキとスペシャル両方の select に同じ `w-40` を適用すること。

## 完了条件
- ほとんどのブキ/スペシャル名が途切れず表示されること
- ブキとスペシャルのドロップダウン幅が揃っていること
- テーブル全体が不必要に幅を取らないこと
