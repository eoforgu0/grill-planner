# チケット管理

## 運用ルール

- 1チケット = 1ファイル（`XXX_短い説明.md`）
- 番号は3桁ゼロ埋め、優先度順にソートされる
- 対応完了後、ファイル名の先頭に `DONE_` を付けてリネーム（例: `DONE_001_fix_label.md`）
- 各チケットの「完了条件」をすべて満たしたら完了とする
- 他チケットに統合された場合、`MERGED_INTO_XXX_` プレフィックスが付く（対応不要）

## 優先度

- 001〜099: バグ修正（仕様不一致）
- 100〜199: 改善（品質・保守性向上）

## 対応手順

1. `tickets/` 内の未完了チケット（`DONE_` も `MERGED_INTO_` も付いていないもの）を番号順に確認
2. 1チケットずつ対応し、完了条件を満たしたらリネーム
3. 関連するテストがあれば実行して通ることを確認

## 統合について

一部のチケットは関連する別チケットの内容を統合している。統合先チケットの冒頭に「## 統合」セクションがあり、何を含んでいるか記載されている。統合元チケットは `MERGED_INTO_XXX_` プレフィックス付きにリネーム済みで、対応不要。
