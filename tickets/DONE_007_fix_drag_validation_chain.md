# 007: ドラッグ中のバリデーション（validateMoveDefeat）にも同じチェーン検証の不備がある

## 種別
バグ（致命的、006 の派生）

## 症状
撃破マーカーをドラッグして、本来は不正な位置（湧いていないタイミング）に配置できてしまう。

## 原因
006 と同根。`validateMoveDefeat` も内部で `validateAllDefeats` → `isConsistentDefeat` を使っており、
チェーン検証の不備がそのまま移動バリデーションにも影響する。

## 修正箇所
006 の修正で `validateAllDefeats` が修正されれば自動的に解消される。追加の修正は不要。

## 完了条件
006 の修正後、以下を確認:
- 撃破マーカーをドラッグして不正な位置（湧きが消費済み）に移動できないこと
- 正当な位置への移動は従来通り可能なこと
- ドラッグ中のリアルタイムバリデーション（isValidPosition 表示）が正しく動作すること

## 備考
このチケットは 006 と同時に完了する。006 の完了確認時に本チケットの完了条件も合わせて確認すること。
